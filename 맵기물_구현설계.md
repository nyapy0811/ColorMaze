# 맵 기물 7종 구현 설계

`ColorMaze 기획서.md` 4장 기준. 이미 완성된 4.2 컬러 필터(`ColorFilterBlock`)를 포함해 7개 기물을 하나의 일관된 구조로 정리한다. 실제 코드 작성 전 설계 단계.

추후 기물이 추가될 것을 감안해, "어떤 효과인가"와 "언제·몇 번 발동하는가"를 분리하는 것을 설계 원칙으로 삼는다.

## 1. 클래스 계층

```
MapObjectBase (abstract, MonoBehaviour, RequireComponent(Collider))
 ├─ FilterBlockBase          판정형(필터) — 4.2, 4.3
 │   ├─ ColorFilterBlock         (컬러 필터)
 │   └─ RgbFilterBlock         (RGB 필터)
 │
 ├─ AcquireObjectBase      획득형 — 4.1
 │   └─ ColorPalette           (컬러 팔레트)
 │
 ├─ ClearObjectBase        클리어형 — 4.7
 │   └─ ColorCanvas            (캔버스)
 │
 └─ ConsumableObjectBase   소모성·1회성(범용) — 향후 스택과 무관한 1회성 기물도 여기서 파생
     └─ StackModifierConsumable   스택을 다루는 소모성 기물 전용 중간 계층 — 4.4, 4.5, 4.6
         ├─ StackChanger            (스택 체인저)
         ├─ ColorChanger            (컬러 체인저)
         └─ Dropper                 (스포이드)
```

`ConsumableObjectBase`를 "스택 변형"과 분리해 둔 이유: 나중에 스택과 무관한 1회성 기물(예: 1회용 발판, 순간이동 패드)이 추가돼도 `ConsumableObjectBase`만 상속하면 되고, `ColorStacks`를 몰라도 된다. 현재 3개 기물처럼 "스택을 조작하는 소모성 기물"에만 `StackModifierConsumable`이라는 한 단계 더 구체적인 중간 클래스를 둔다.

## 2. MapObjectBase — 공통 루트

모든 기물이 공유하는 것만 담는다.

- `[RequireComponent(typeof(Collider))]`
- `protected Collider Col { get; }` — Awake에서 캐싱
- `protected ColorStacks Player { get; }` — 지연 탐색 후 캐싱(현재 코드베이스에 별도 Player 태그 컴포넌트가 없어 `ColorStacks` 보유 오브젝트를 플레이어로 간주하는 기존 관례를 그대로 따름)
- `protected bool IsPlayer(Collider other) => Player != null && other.GetComponentInParent<ColorStacks>() == Player;` — 4개 그룹 모두에서 반복되는 "이 콜라이더가 플레이어인가" 판정을 한 곳으로 모음

이 공통화 덕분에 각 그룹 베이스는 "무엇을 할지"만 신경 쓰면 된다.

## 3. 그룹별 베이스 클래스

### FilterBlockBase (판정형 — 4.2, 4.3)
- `abstract bool Matches(ColorStacks player);`
- `Update()`: 매 프레임 `Matches(Player)` 결과로 `Col.isTrigger` 토글(일치 시 트리거=통과 허용, 불일치 시 솔리드)
- `OnTriggerEnter/Exit`: `IsPlayer(other)`로 진입·완전 통과 추적, 통과 완료 시 `Player.ResetAll()`
- 외형(공용 머티리얼·투명도 적용, 메시 병합 연동)은 `FilterBlockBase`에 내장(별도 매니저 컴포넌트 없음)
- 하위 클래스는 `Matches()`만 구현:
  - `ColorFilterBlock.Matches` = 정확한 RGB 일치
  - `RgbFilterBlock.Matches` = `Player.GetMaxColors()`에 지정 색이 포함되는지

### AcquireObjectBase (획득형 — 4.1)
- `abstract void OnAcquire(ColorStacks player);`
- `OnTriggerEnter`: `IsPlayer(other)`면 `OnAcquire(Player)` 호출. 자기 파괴 없음 → 반복 발동 가능
- `ColorPalette.OnAcquire` = 지정된 R/G/B만큼 `player.Add(color, amount)`

### ClearObjectBase (클리어형 — 4.7)
- `public bool Completed { get; private set; }`
- `abstract bool CheckCondition(ColorStacks player);`
- `abstract void OnCompleted();` — 완료 시 외형 변경 등
- `OnTriggerEnter`: `!Completed && IsPlayer(other) && CheckCondition(Player)`면 `Completed = true` 후 `OnCompleted()` 호출 + `EventBus.Publish(new CanvasCompleted{...})`
- 한 번 완료되면 계속 유지(잠금) — 순차 완료 방식(이전 설계에서 확정)
- `ColorCanvas.CheckCondition` = 세 스택 값이 목표값과 정확히 일치하는지

### ConsumableObjectBase (소모성·1회성 — 범용)
- `bool Consumed { get; private set; }`
- `abstract void Apply();` — 어떤 자원(스택 등)을 다루는지 모름, 그냥 "발동"만 정의
- `OnTriggerEnter`: `Consumed`면 무시(중복 발동 방지), `IsPlayer(other)`면 `Apply()` 호출 후 `Consume()`
- `protected virtual void Consume()`: `Consumed = true`, `Col.enabled = false`, `Destroy(gameObject)` (씬에 직접 배치되는 오브젝트라 풀링 대상 아님 — 필요해지면 `PoolManager.Despawn`으로 교체 가능)

#### StackModifierConsumable : ConsumableObjectBase (4.4~4.6 중간 계층)
- `abstract void ApplyToStacks(ColorStacks player);`
- `Apply() override => ApplyToStacks(Player);`
- 하위 클래스:
  - `StackChanger.ApplyToStacks`: `a=Get(colorA); b=Get(colorB); SetValue(colorA,b); SetValue(colorB,a);`
  - `ColorChanger.ApplyToStacks`: `max=세 스택 중 최댓값; 각 색 c에 대해 SetValue(c, max-Get(c));`
  - `Dropper.ApplyToStacks`: `SetValue(targetColor, 0);`

## 4. ColorStacks에 추가할 API

- `SetValue(LightColor c, int value)` — 절대값 지정(내부적으로 `Wrap` 적용) 후 변경 시 이벤트 발행. `Add`도 내부적으로 이를 재사용하도록 정리 가능.
- `GetMaxColors()` — 세 스택 중 최댓값을 가진 색상 목록(동률 포함) 반환. RGB 필터 판정에 사용.

## 5. 새 이벤트

- `CanvasCompleted : IEvent` — 캔버스 하나가 조건을 만족했을 때
- `StageCleared : IEvent` — 스테이지 내 모든 `ColorCanvas`가 완료됐을 때, `LevelManager`가 구독해 클리어 처리

## 6. 구현 순서

1. `ColorStacks.SetValue`, `GetMaxColors` 추가
2. `MapObjectBase` 작성(공통 Player/IsPlayer 로직)
3. `FilterBlockBase` 추출 → 기존 `ColorFilterBlock` 이관 → `RgbFilterBlock` 구현
4. `ConsumableObjectBase` → `StackModifierConsumable` → `StackChanger`/`ColorChanger`/`Dropper` 구현
5. `AcquireObjectBase` → `ColorPalette` 구현
6. `ClearObjectBase` → `ColorCanvas` 구현 + `CanvasCompleted`/`StageCleared` 이벤트 + `LevelManager` 연동

## 7. 향후 기물 추가 시 참고

- 판정형(통과 조건)만 다른 기물 → `FilterBlockBase` 상속, `Matches()`만 구현
- 반복 획득형(효과만 다른 기물) → `AcquireObjectBase` 상속, `OnAcquire()`만 구현
- 클리어 조건만 다른 기물 → `ClearObjectBase` 상속, `CheckCondition()`만 구현
- 스택과 무관한 1회성 기물(예: 1회용 발판) → `ConsumableObjectBase` 직접 상속, `Apply()`만 구현
- 스택을 다루는 새 소모성 기물 → `StackModifierConsumable` 상속, `ApplyToStacks()`만 구현
